# UI/UX Plan: 공공 실거래가 시세 자동 조회

Status: READY_FOR_DIRECTOR_REVIEW

## User Goal

사용자는 계약 주소와 주택 유형을 바탕으로 공공 실거래가를 조회하고, 근거와 한계를 확인한 뒤 분석에 사용할 시세 입력값으로 적용할 수 있어야 한다.

## Primary Flow

1. 사용자가 계약 기본 정보를 저장한다.
2. 시세 입력 카드에서 `공공 실거래가 조회`를 누른다.
3. 화면은 조회 중 상태를 표시하고 기존 입력값은 유지한다.
4. 조회 성공 시 추정 매매가, 전세 참고 금액, 표본 수, 기준월 범위, 출처, 경고를 preview로 표시한다.
5. 사용자가 `조회 결과 적용`을 누르면 기존 시세 입력 field가 채워진다.
6. 사용자가 기존 `시세 저장` 버튼으로 최종 저장한다.
7. 분석 실행 후 리포트에서 계산값과 출처를 확인한다.

## Screens Or Interfaces

- 기존 단일 workspace의 `시세 입력` 카드
- 새 조회 버튼, 조회 결과 preview, 경고/오류 메시지
- 기존 분석 및 결과 카드

## States

- Loading: `공공 실거래가 조회 중...`
- Empty: 계약 생성 전에는 조회 버튼 비활성화
- Success: 조회 후보값, 표본 수, 기준월, 출처, 적용 버튼 표시
- Error: 주소 해석 실패, 표본 부족, provider 비활성화, 외부 API 오류를 카드 안에 표시
- Permission denied: 기존 `X-User-Id` 접근 제어 오류 메시지 유지

## Copy And Messaging

- 조회 버튼: `공공 실거래가 조회`
- 적용 버튼: `조회 결과 적용`
- 표본 부족: `유사 실거래 표본이 부족해 계산 기준값으로 확정하지 않았습니다.`
- 주소 불명확: `주소를 시군구/법정동으로 해석하지 못했습니다. 주소를 보완하거나 수동 입력을 사용해 주세요.`
- provider 비활성화: `공공 실거래가 조회가 설정되어 있지 않습니다. 수동으로 시세를 입력할 수 있습니다.`

## Simple Feature Checklist

- [ ] 조회 전 기존 시세 입력값을 덮어쓰지 않는다.
- [ ] 조회 결과의 출처와 기준월을 사용자에게 표시한다.
- [ ] 적용 후에도 저장은 명시적 `시세 저장`으로만 수행한다.
- [ ] 조회 실패 시 수동 입력 fallback이 명확하다.
- [ ] 모바일/데스크톱에서 시세 카드 텍스트가 넘치지 않는다.

## Developer Notes

- lookup API는 저장하지 않는 read-like command다.
- frontend는 공공 API key를 알면 안 된다.
- public source는 확정 시세가 아니라 실거래 기반 참고값으로 표현한다.

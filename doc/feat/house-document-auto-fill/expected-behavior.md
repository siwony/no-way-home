# Expected Behavior

## Primary Flow

1. 사용자가 기존 진단 화면에서 문서 자동 입력을 선택한다.
2. 등기부등본과 임대차 계약서를 업로드한다.
3. 시스템은 문서를 저장하고 처리 상태를 `UPLOADED -> EXTRACTING -> REVIEW_REQUIRED`로 갱신한다.
4. 사용자는 자동 추출된 필드와 원문 근거를 확인한다.
5. 사용자는 필드별로 수정하거나 제외할 수 있다.
6. 사용자가 승인하면 승인된 필드만 기존 계약 정보/등기 수기 확인/시세 관련 입력 영역에 반영된다.
7. 분석 실행 전 화면에는 자동 입력 출처와 검토 필요 항목이 표시된다.

## Document States

- `UPLOADED`: 원본 저장 완료
- `EXTRACTING`: OCR 또는 텍스트 추출 진행 중
- `REVIEW_REQUIRED`: 추출 결과가 있고 사용자 검토가 필요함
- `APPROVED`: 사용자가 반영을 승인함
- `FAILED`: 처리 실패
- `DELETED`: 사용자 또는 retention 정책으로 원본 삭제됨

## Confidence Behavior

- 각 추출 필드는 `value`, `sourceDocument`, `sourcePage`, `sourceText`, `confidence`, `reviewStatus`를 가진다.
- 낮은 신뢰도 필드는 자동 반영하지 않고 사용자가 직접 확인해야 한다.
- 주소/임대인/소유자/금액 같은 핵심 필드는 신뢰도가 높아도 최종 승인 전까지 편집 가능해야 한다.

## Mismatch Behavior

- 계약서 임대인과 등기부등본 소유자가 다르면 `확인 필요`로 표시한다.
- 계약서 주소와 등기부등본 주소가 다르면 주소 정규화 실패 가능성과 실제 불일치 가능성을 함께 표시한다.
- 등기부등본 발급일이 오래되면 최신 등기부등본 재확인을 안내한다.
- 특약 문장이 추출되면 위험을 단정하지 않고 검토 후보로 표시한다.

## Error Behavior

- 암호화된 PDF, 이미지 품질 부족, OCR 실패, 문서 종류 오분류, 지원하지 않는 문서 형식을 구분해 안내한다.
- 추출 실패해도 사용자는 기존 수기 입력 흐름으로 계속 진행할 수 있다.
- 외부 provider 장애는 retry 가능 상태로 표시하고, 실패 로그에 민감 원문을 남기지 않는다.

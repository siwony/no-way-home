# Expected Behavior

## Inputs

입력값은 목적, 검증 조건, 없을 때 처리를 명확히 가진다. 주민등록번호는 입력받지 않는다. 임대인 이름은 최소 수집하고 저장 시 암호화 대상으로 본다. 주소는 도로명주소와 지번주소를 모두 고려한다.

필수 금액 값은 0 이상이어야 한다. 보증금, 월세, 계약일, 입주일, 계약 유형, 주택 유형이 유효하지 않으면 요청을 거절한다.

## External Data Handling

- 등기부등본이 없으면 핵심 위험도 판정 불가 사유를 추가한다.
- 건축물대장이 없으면 위반건축물 및 용도 확인 불가 사유를 추가한다.
- 시세 정보가 없으면 전세가율과 총 위험 노출 비율 계산 불가 사유를 추가한다.
- 각 외부 자료는 출처, 기준일, 발급일 또는 조회일을 저장한다.

## Registry Analysis

등기부등본은 표제부, 갑구, 을구로 분석한다. 주소, 건물 종류, 면적, 대지권, 현재 소유자, 소유권 이전 이력, 압류, 가압류, 가처분, 경매개시결정, 신탁등기, 근저당권, 전세권, 임차권등기, 채권최고액, 권리자, 접수일자, 말소 여부를 확인한다.

말소된 권리는 현재 유효 권리와 분리한다. 접수일자를 기준으로 사용자 보증금보다 선순위인지 판단한다. 근저당권은 채권최고액 기준으로 계산한다. 신탁등기는 임대 권한 확인 필요로 표시한다.

## Building Ledger Analysis

건축물대장은 건축물 용도, 주거용 여부, 근린생활시설 여부, 위반건축물 여부, 호실 존재 여부, 계약 면적과 공부상 면적 일치 여부, 사용승인일, 다가구/다세대/오피스텔 여부를 분석한다.

근린생활시설을 주거용으로 계약하는 경우, 위반건축물인 경우, 다가구주택의 선순위 임차인 보증금이 확인되지 않는 경우, 호실이 명확하지 않은 경우는 별도 위험 사유로 표시한다.

## Risk Metrics

전세가율:

```text
전세가율 = 보증금 / 추정 시세 * 100
```

총 위험 노출 비율:

```text
총 위험 노출 비율 = (선순위채권 + 보증금) / 추정 시세 * 100
```

보수적 회수 시뮬레이션:

```text
예상 낙찰가 = 추정 시세 * 예상 낙찰가율
사용자 회수 가능 금액 = min(보증금, max(0, 예상 낙찰가 - 선순위채권))
예상 부족액 = min(보증금, max(0, 보증금 - 사용자 회수 가능 금액))
```

사용자 회수 가능 금액은 음수가 될 수 없고 보증금을 초과할 수 없다. 예상 부족액도 `0` 이상 `보증금` 이하로 제한한다.

예상 낙찰가율이 없으면 보수적 기본값을 사용하거나 “회수 시뮬레이션 계산 기준 부족”으로 표시한다.

## Risk Levels

최종 위험도 우선순위는 `CRITICAL > DANGER > CAUTION > SAFE`이다.

- `CRITICAL`: 경매개시결정, 압류, 총 위험 노출 비율 100% 이상, 소유자 불일치 및 위임 미확인
- `DANGER`: 가압류, 가처분, 임차권등기, 신탁등기, 위반건축물, 총 위험 노출 비율 80% 이상, 전세가율 80% 이상
- `CAUTION`: 근저당권, 전세가율 70% 이상, 시세 정보 부족, 건축물대장 미확인, 다가구 선순위 임차인 미확인, 신축빌라 시세 신뢰도 낮음
- `SAFE`: 중대 위험 신호가 없고 등기상 소유자 일치, 압류/가압류/경매/임차권등기 없음, 건축물대장상 주거용, 위반건축물 아님

## Report

리포트는 종합 위험도, 한 줄 요약, 기준 시점, 핵심 위험 사유 최대 5개, 등기 분석 결과, 건축물 분석 결과, 보증금 위험 분석, 보수적 회수 시뮬레이션, 추가 확인 필요 항목을 포함한다.

추가 확인 항목에는 계약 직전 등기 재열람, 건축물대장 확인, 다가구 선순위 임차인 확인, 전입신고/확정일자 안내, 보증보험 가입 가능 여부 확인을 포함한다.

## API Behavior

- `POST /api/house-checks`: 진단 요청 생성
- `POST /api/house-checks/{checkId}/registry-file`: 등기부등본 업로드
- `POST /api/house-checks/{checkId}/building-ledger-file`: 건축물대장 업로드
- `POST /api/house-checks/{checkId}/market-price`: 시세 정보 입력 또는 조회 결과 저장
- `POST /api/house-checks/{checkId}/analyze`: 위험도 분석 실행
- `GET /api/house-checks/{checkId}/report`: 리포트 조회
- `GET /api/house-checks/{checkId}/checklist`: 체크리스트 조회

주요 에러 코드는 `HOUSE_CHECK_NOT_FOUND`, `INVALID_CONTRACT_TYPE`, `INVALID_DEPOSIT_AMOUNT`, `REGISTRY_FILE_REQUIRED`, `MARKET_PRICE_NOT_AVAILABLE`, `ANALYSIS_NOT_READY`, `ACCESS_DENIED`를 사용한다.
